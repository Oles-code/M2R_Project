## 1. Overview

DirectLiNGAM defines edges from a single point-estimate adjacency matrix; the bootstrap is currently used only to annotate those edges with a selection frequency. This study asks whether a different way of *aggregating* the bootstrap would select a better edge set. We compare four selection rules and a hyperparameter sweep against a fixed interventional ground truth: an edge cause→effect is **confirmed** iff knocking down the cause produces a powered, BH-significant Mann–Whitney shift in the effect (the union of the `confirmed` and `cyclic` validation verdicts). Because the ground truth is held fixed, differences in confirmation rate reflect the *selection rule*, not the validation. The bootstrap is run to 200 resamples once; n=50/100 are leading slices of the same run.

## 4. Cyclic detection heuristic

For each unordered pair {A, B} we count, across the 100 bootstrap adjacencies, how many resamples placed A→B versus B→A, and flag the pair as *bootstrap-cyclic* when BOTH directions appear in at least `min_frac` of resamples. The interventional validation independently labelled **246** unordered pairs `cyclic` (both directions BH-significant).

| min_frac | Pairs flagged | Overlap w/ interventional cyclic | False positives | Recall of interventional cyclic |
|---|---|---|---|---|
| 0.20 | 198 | 128 | 70 | 0.520 |
| 0.30 | 115 | 76 | 39 | 0.309 |
| 0.40 | 47 | 30 | 17 | 0.122 |

**Caveat — this is a heuristic, not a test.** DirectLiNGAM is derived under acyclicity, so reading ordering-instability across resamples as evidence for a cycle steps outside the model's theoretical guarantees. A pair can flip direction across resamples for reasons unrelated to feedback (near-tied independence scores, finite-sample noise under the heavy tails of this data). The interventional validation remains the gold standard for cyclicity; the bootstrap flag is best read as a **complementary diagnostic for potential feedback loops** worth examining, not a verdict.

## 5. Recommendations

- The baseline `point_estimate` selects 335 edges at a 0.839 confirmation rate — high recall, low precision (it keeps every non-zero coefficient, including resampling artefacts).

- The best precision/recall trade-off among the aggregation rules that keep ≥ 10 edges is **edge_frequency @ 0.60**: 203 edges at a 0.892 confirmation rate. Frequency- and directional-stability thresholds concentrate the edge set on reproducible structure, which is what the interventional data rewards.

- For the report we keep the point-estimate edge set (it is the model's actual output and preserves recall for the false-omission analysis) but annotate every edge with bootstrap frequency, and we use the directional/frequency thresholds above as a stability filter when a higher-precision sub-graph is wanted.
